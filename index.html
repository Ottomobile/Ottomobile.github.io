UBC Snowbots - Pylon Detection Algorithm

Implemented a pylon detection algorithm for the team’s autonomous ground robot using C++, OpenCV, and the Robot Operating System.

Drawing Rectangles that Bound Related Regions

Used rectanges' midpoints to determine related regions that identify a pylon

 

UBC Snowbots is an engineering design team that makes an autonomous ground vehicle to compete in the international Intelligent Ground Vehicle Competition (IGVC) held in Michigan. Our vehicle nagivates a terrain of obstacles without any human input.

I was responsible for implementing pylon detection on the robot. The goal was to identify the orange and white striped pylons from the camera input stream and frame them with a large black box.

Details of algorithm:

  1. The algorithm first uses colour filtering to identify orange regions in an input stream that indicate the locations of pylons.
  2. Then, contours from these filtered images are extracted and used to draw rectangles that bound the orange regions.
  3. The midpoints of the rectangles are analysed to determine the proximity of the orange regions.
  4. If two rectangles are close to each other, those orange regions should be from the same pylon and bound a white region in between. Finally, draw a large black box that covers those orange and white regions to mark the pylon.

Gmail Webpage Retriever

Constructed an application that processes incoming emails containing Google search keywords or webpage addresses and automatically sends back an email with the attached search results or webpage content.

Github Repo

Expand details

This Java application enables a user to browse the internet through their email.

Four different retrieval methods are supported:

  1. Searching Google through Selenium – User sends an email with Google search keywords and automatically receives an email back containing the Google search results. The search results contain webpage addresses that could be used with the retrieval methods below.

  2. Downloading a webpage as a PDF through Selenium – User sends an email containing a webpage address and automatically receives an email back with the webpage attached as a PDF.

  3. Extracting webpage’s HTML source – User sends an email containing a webpage address and automatically receives an email back containing the plain HTML source of the webpage.

  4. Mirroring website content using HTTrack – User sends an email containing a webpage address and HTTrack options and automatically receives a zip file containing the webpage contents. This method allows the retrieval of HTML files directed by the links found in the original page.

Arduino Robot

As a team, built a robot that could autonomously avoid objects and also be controlled by a Wii Nunchuk Controller.

Course Work

Expand details

Components and connections of the robot

Robot Controlled by Wii Nunchuk

Wheel Encoder

Marked wheel for the wheel encoder

 

The goal of this project was to construct an autonomous vehicle that moves straight and turns 90 degrees to avoid objects. Our team’s Arduino-controlled robot contained an ultrasonic sensor mounted on a servo motor for object detection. The robot starts moving forward until it detects an object in front of it. Then, the robot stops and the servo motor rotates the ultrasonic sensor. Whichever direction has the most space to move will be the direction the robot will travel towards.

A Wii Nunchuk Controller could also control the robot’s movement. Tilting the Nunchuk controls the speed and direction of the robot. Moving the joystick rotates the servo motor with the ultrasonic sensor. Pressing the Z-button allowed the user to switch between autonomous and human-controlled modes.

A challenge we faced was getting the robot to go straight and turn exactly 90 degrees. Due to battery drainage, friction, and weight balance, the robot’s movements over time would become inconsistent. Wheel encoders were designed to address this issue by using an IR sensor to record wheel rotations and adjusting the speed of the motors to compensate for rotation variations.

Our team built a software application and a physical piano interface that could communicate with each other over wifi. Clicking the screen or pressing keys on the computer could wirelessly trigger the physical piano to play sounds. Similarly, pressing the keys on the piano could wirelessly play sounds and update the user interface of the software application. The software application also allowed the recording and playback of melodies created using either interfaces.
